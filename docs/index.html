// the problem

You open GitHub. 47 open PRs. Three of them are from a bot. One is a 64-file migration someone pushed at 4:47 PM on a Friday. You've been tagged as a reviewer on all of them. Your coffee is cold. Your will to live is room temperature.

You click into the first PR. 800 lines changed. No description. The branch name is fix-stuff-v3-final-FINAL. You scroll through the diff for 20 minutes. You leave a comment that says "LGTM" because you have lost the ability to feel.

RevBuddy exists because this is not okay.

It watches your repos. It analyzes every PR automatically — summary, deep code review, test plan, ticket alignment. It scores priority so you know what's actually on fire. And it gives you a terminal UI with an AI chat agent that has actually read the code so you can ask it "what's the risk here?" instead of squinting at diffs until your eyes bleed.

// what it does

auto-analysis

PRs are analyzed the moment they appear. 4 AI analyzers run in parallel with configurable concurrency. No manual triggering.

priority scoring

Critical security findings? Top of the queue. Tiny 2-file PR? Prioritized over the 800-line monstrosity. Old PRs get age boosts. Labels map to priority.

interactive review

Split-pane TUI: artifacts on the left, AI chat on the right. Claude runs inside the PR's worktree with full file access. Ask it anything.

streaming chat

Token-by-token streaming. See thinking blocks, tool invocations, and results in real-time. It's like pair-reviewing with someone who actually read the PR.

persistent state

Repos, queue items, artifacts, and chat history survive server restarts. Your progress is never lost.

real-time updates

WebSocket pushes queue changes, analysis progress, and chat streams. No polling, no refresh, no stale data.

// how it works
